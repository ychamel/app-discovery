### `UX-005`: No way for developers to preview their app page before approval

- **Reporter:** User (Developer)
- **Date Reported:** 2026-06-29
- **Priority:** Medium
- **Status:** `TRIAGED`
- **Patch/Feature Slug:** `patch-developer-app-preview` (queued)

#### UX Observation

After submitting an app, developers have no way to preview how their app page will appear to users while it is still pending approval. The app page (`apps/pages/`) is only publicly accessible once the app has `ACCEPTED` status; there is no preview surface for the submitter.

#### Impact & User Friction

- Developers cannot verify that their description, screenshots, and metadata will render correctly before the app goes live.
- Any formatting or presentation issues are only discoverable post-approval, requiring an edit cycle with no guarantee of re-approval timing.
- Reduces confidence in the submission experience and increases avoidable correction requests.

#### Suggested Improvement

Provide the submitting developer with a **preview link** to their own app page that works regardless of approval status — visible only to the app's owner (and admins). This could be:

- A "Preview as user" link on the My Submissions page (`catalog:my-apps`) and/or on the submission detail/edit page, pointing to the existing app page view but bypassing the ACCEPTED gate for the owner.
- A dedicated read-only preview route, or an owner-aware bypass on the existing `app_page` view (e.g., `request.user == app.developer` skips the status gate).

#### Scope Note

This likely requires a change to the app page access gate in `apps/pages/` (currently enforces `ACCEPTED` status) and may touch `apps/catalog/`. If a migration or new endpoint is needed, this escalates to the Feature Track.
