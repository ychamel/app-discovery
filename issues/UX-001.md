### `UX-001`: Mobile touch target sizes for catalog search filters are too small

- **Reporter:** QA Team / UX Designer
- **Date Reported:** 2026-06-28
- **Priority:** Medium
- **Status:** `TRIAGED`
- **Patch/Feature Slug:** `TBD` (Patch Track — responsive CSS only; queued behind UX-003)

#### UX Observation
On mobile viewports (widths < 640px), the interest tags and category filters on the catalog search page `/discover/` are rendered with standard browser spacing. The interactive clickable area for each tag is approximately 24px in height.

#### Impact & User Friction
According to mobile accessibility guidelines (WCAG 2.1 Success Criterion 2.5.5), touch targets should be at least 44x44 CSS pixels. The current size of 24px leads to misclicks, particularly on mobile/tablet devices where users struggle to tap specific subcategories without accidentally selecting adjacent ones.

#### Suggested Improvement
1. Increase padding inside each filter/tag pill on mobile viewports to at least `0.5rem 1rem`.
2. Add a minimum height of `44px` or adjust margins to ensure touch targets do not overlap.
3. Leverage CSS `:active` scale/color transitions to provide clear haptic/visual feedback on touch devices.

#### Visual References / Markup
File: `apps/discovery/templates/discovery/catalogue.html`
CSS Class: `.filter-pill`, `.tag-chip`
Target selector:
```css
@media (max-width: 640px) {
  .tag-chip {
    padding: 0.5rem 1rem;
    min-height: 44px;
    display: inline-flex;
    align-items: center;
  }
}
```
