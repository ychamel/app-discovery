### `UX-003`: No navigation path to view and manage pending/withdrawn app submissions

- **Reporter:** User
- **Date Reported:** 2026-06-28
- **Priority:** High
- **Status:** `RESOLVED` (2026-06-28, released local/dev — see [RELEASE_NOTES.md](../patches/patch-developer-submissions-nav/RELEASE_NOTES.md))
- **Patch/Feature Slug:** [`patch-developer-submissions-nav`](../patches/patch-developer-submissions-nav/) (Patch Track — template/nav only, no schema/API/ADR)

#### UX Observation
When a user with the `developer` role submits a new application, they are redirected to the app detail page (e.g. `/apps/{id}`). However, if they navigate away from this page, there is no visible way in the user interface to return to their submission or monitor its progress.

Specifically:
1. The global header navigation (`apps/core/templates/core/base.html`) does not provide a link to the submissions list page (`/apps`, mapped to `catalog:my-apps`) or the dashboard for logged-in developers.
2. The Developer Dashboard (`/dashboard/`, mapped to `dashboard:my-apps`) only displays telemetry for *accepted* apps. If a developer has only pending, withdrawn, or rejected submissions, they are shown an empty state ("No accepted apps yet") with no links or navigation options leading to their actual pending submissions.

As a result, the views/templates that allow checking application status, editing details, uploading screenshots, or withdrawing/resubmitting applications (at `/apps` and `/apps/{id}`) become virtually unreachable.

#### Impact & User Friction
- **Inability to Track Status:** Developers cannot check if their application is pending, rejected, or needs changes.
- **Inability to Edit/Withdraw:** Developers cannot perform lifecycle actions (such as editing details, adding screenshots, or withdrawing/resubmitting) because they cannot find the details page for their submission.
- **Dead Ends:** The Developer Dashboard presents a dead end for users whose applications have not yet been accepted, rather than guiding them to manage their active submissions.

#### Suggested Improvement
1. **Header Navigation Links:** Update the global header in `apps/core/templates/core/base.html` to conditionally display links to "My Submissions" (or "Developer Dashboard") when the logged-in user holds the `developer` role.
2. **Dashboard Navigation:** Add navigation options (e.g., tabbed navigation) on the Developer Dashboard to let users switch between "Analytics/Telemetry" (available for accepted apps) and "Submissions" (available for all submitted apps, pointing to `catalog:my-apps`).
3. **Empty-State Call to Action:** Update the empty state on the Developer Dashboard to include a link to "View my submissions" or "Manage pending submissions" alongside the existing "Submit an app" button.
