### `BUG-002`: Profile display name update fails with 405 Method Not Allowed and idles

- **Reporter:** User / Developer
- **Date Reported:** 2026-06-28
- **Severity:** High
- **Status:** `NEW`
- **Patch/Feature Slug:** `TBD`

#### Description & Impact
When an authenticated user navigates to their profile page (`/profile`) and attempts to update their display name, the form submission fails. The server logs show a `405 Method Not Allowed` returned by the `/me` endpoint.

Because the form submission is boosted by HTMX (`hx-boost="true"` on the parent `<main>` container) and the server returns a 405 error status code, HTMX's default behavior is to do nothing, leaving the user with no visual feedback or error message. The page appears to "idle" or hang indefinitely.

#### Steps to Reproduce
1. Log in and navigate to the profile page `/profile`.
2. Locate the "Edit display name" section under "Account Information".
3. Enter a new display name in the input field.
4. Click the "Save" button.
5. Observe that the page does not refresh, no success/error message is displayed, and the form appears to idle.
6. Verify the browser's Network tab or server logs, which show a `POST /me` request returning a `405 Method Not Allowed` status.

#### Expected Behavior
- The display name should be successfully updated on the server and the UI should reflect the change.
- If the submission fails, an appropriate error message should be displayed to the user instead of idling silently.

#### Actual Behavior & Details
- The client submits a `POST` request to `/me` because standard HTML forms only support `GET` and `POST`.
- The endpoint is handled by `MeView` in `apps/accounts/views.py`, which is a DRF `APIView` that only implements `get()`, `patch()`, and `delete()` methods. It does not implement `post()`.
- Consequently, Django/DRF returns `405 Method Not Allowed`.
- Due to `hx-boost="true"` on the parent container, HTMX intercepts the form's post request. Since HTMX does not process or swap error responses (like 405) by default, the interface remains unchanged with no user feedback.

Server log snippet:
```
[28/Jun/2026 17:57:11] "POST /me HTTP/1.1" 405 40
Method Not Allowed: /me
```
Form definition in `apps/accounts/templates/accounts/profile.html`:
```html
<form method="post" action="{% url 'accounts:me' %}" data-method="patch" class="stack" style="--gap: var(--space-3); border-top: 1px solid var(--color-border); padding-top: var(--space-4);">
```
View definition in `apps/accounts/views.py`:
```python
class MeView(APIView):
    permission_classes = [IsAuthenticated]

    def get(self, request):
        return Response(AccountSerializer(request.user).data)

    def patch(self, request):
        # ...
```
