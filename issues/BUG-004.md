### `BUG-004`: "Try App" button links to the platform URL instead of the app's own URL

- **Reporter:** User
- **Date Reported:** 2026-06-29
- **Severity:** High
- **Status:** `NEW`
- **Patch/Feature Slug:** `TBD`

#### Description & Impact
On the app detail page, the **"Try App"** (or equivalent CTA) button is the primary conversion action — it should navigate the user directly to the external URL of the listed app. Instead, it currently navigates to the platform's own URL (i.e., it is a self-referential or misconfigured link). This completely breaks the conversion flow: users who click "Try App" go nowhere useful, and developers get zero click-through from the listing.

#### Steps to Reproduce
1. Log in (or browse as anonymous).
2. Open any approved app's detail page.
3. Click the **"Try App"** / **"Visit App"** button.
4. Observe the destination URL in the address bar.

#### Expected Behavior
The browser navigates to the `app.url` (or equivalent field) that the developer provided during submission — i.e., the live external URL of the app being listed.

#### Actual Behavior & Details
The button navigates to the platform's own base URL (or a platform-internal route) rather than the app's external URL. The `href` attribute of the button/anchor is likely hardcoded to `"/"` or `request.get_host()` instead of being bound to `app.url`.

Check the template rendering the app detail CTA — look for the anchor tag associated with "Try App" and verify it interpolates `{{ app.url }}` (or the correct model field name).
