### `BUG-000`: App detail page profile link broken when anonymous

- **Reporter:** QA Team
- **Date Reported:** 2026-06-28
- **Severity:** Medium
- **Status:** `RESOLVED`
- **Patch/Feature Slug:** [`patches/patch-anonymous-profile-link/`](../patches/patch-anonymous-profile-link/)

#### Description & Impact
When a user is not logged in (anonymous session) and navigates to an app's detail page, clicking on the publisher's username or profile link results in a `500 Internal Server Error` instead of redirecting to the login page or displaying a public profile.

#### Steps to Reproduce
1. Log out or open an incognito window.
2. Navigate to `/discover/` and click on any app page.
3. Scroll to the "Publisher" section and click the link on the publisher name.

#### Expected Behavior
The publisher's profile should render in read-only mode, or if authentication is required, the user should be redirected to the login page `/accounts/login/` with a `next` query parameter.

#### Actual Behavior & Details
The page displays a standard Django 500 error page.
Server log:
```
AttributeError: 'AnonymousUser' object has no attribute 'profile'
In apps/accounts/views.py at line 42:
  profile = request.user.profile
```
