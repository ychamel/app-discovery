### `UX-002`: App registration tags selection is overwhelming and niche selection is difficult

- **Reporter:** Developer / User
- **Date Reported:** 2026-06-28
- **Priority:** Medium
- **Status:** `NEW`
- **Patch/Feature Slug:** `TBD`

#### UX Observation
When submitting a new app (at `/submit`) or editing details of an existing submission (at `/apps/{id}`), the developer is presented with a large flat list of checkboxes for tags. The selection is extensive, has no visual categorization, no search filter, and no recommendations based on the app's metadata.

#### Impact & User Friction
- **High Cognitive Load:** Scanning a dense list of unstructured tags requires significant developer effort, causing selection fatigue.
- **Difficulty Finding a Niche:** Without search or category splits, finding specific, niche tags is difficult, leading to developers failing to tag their apps accurately.
- **Suboptimal Curation:** Misaligned or incomplete tags directly impact app discovery, search relevance, and recommendation accuracy on the platform.

#### Suggested Improvement
Consider implementing one or more of the following solutions:
1. **Dynamic Custom Tags:** Add an input field to let developers add custom tags that are dynamically adjusted or updated.
2. **Interactive Search Bar:** Add a search box that dynamically filters the visible tag list as the user types.
3. **Smart Tag Recommendations:** Provide recommended tag suggestions based on the name and description text entered by the user (e.g. using lightweight client-side keyword matching or an API recommendation call).
4. **Tag Selection Wizard (Steam-style):** Guide developers through a step-by-step wizard where tags are categorized into distinct groups (e.g., Platforms, Features, Target Audience) with a configurable maximum selection limit per category.
