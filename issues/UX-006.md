### `UX-006`: No direct link from "My Submissions" to the public app page post-approval

- **Reporter:** User (Developer)
- **Date Reported:** 2026-06-29
- **Priority:** Medium
- **Status:** `NEW`
- **Patch/Feature Slug:** `TBD`

#### UX Observation

After an app is approved (`ACCEPTED`), the developer cannot navigate to the public app page directly from their "My Submissions" list (`catalog:my-apps`). They must manually find the app in the Discover / search section to view it as a regular user would.

#### Impact & User Friction

- Extra navigation friction: approved developers have no obvious "View your live page" call to action.
- Breaks the developer workflow loop — submitting and then validating the live page requires a separate discovery search.
- The My Submissions page ([`catalog/my_apps.html`](../apps/catalog/templates/catalog/my_apps.html)) is the natural hub for a developer's apps, but it is currently submission/status-management only, with no link to the live consumer-facing experience.

#### Suggested Improvement

On the My Submissions page (`catalog:my-apps`), for each app with `ACCEPTED` status, surface a **"View as user" / "View live page"** link pointing to the public app page (`pages:app-page` or equivalent). This should be:

- A simple anchor added to the approved app card/row on `catalog/my_apps.html`.
- Conditionally shown only when `app.status == ACCEPTED` (the page is publicly accessible).
- No new route, model change, or migration — template-only addition.

#### Scope Note

This is expected to be a Patch Track item (template-only, no schema/API change). The existing `pages:app-page` URL pattern already serves the public page. The `catalog:my-apps` view already receives the developer's app list with status. All the needed data is in scope.
